# 修复TTS朗读时机逻辑

## 当前问题分析
当前逻辑：
- 加载句子时立即朗读
- 切换下一句时先夸奖再朗读

用户期望的逻辑：
1. 进入input界面时才开始朗读当前句子
2. 翻页下一句时再朗读
3. 正确完成一句时表扬
4. 输入错误时鼓励

## 修改方案

### 1. 修改next_sentence() - 不在加载时朗读
- 移除speak_praise()调用
- 移除speak_sentence()调用
- 只设置句子内容，不进行语音

### 2. 修改handle_input() - 切换到playing状态时开始朗读
- 在用户选择关卡开始游戏时
- 添加开始朗读的逻辑
- 读取当前句子

### 3. 修改next_sentence() - 切换下一句时朗读
- 切换到下一句时朗读新句子
- 不再先夸奖

### 4. 修改表扬逻辑
- 在完成句子时（提交时）表扬
- 使用speak_praise()

### 5. 保持错误鼓励
- 在错误输入时使用speak_encouragement()
